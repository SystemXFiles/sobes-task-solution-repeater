# sobes-task-solution-repeater

Данный репозиторий хранит решение задачи, которую мне дали на собеседовании.

## Задача

Реализовать микросервис с двумя эндопинтами:

1. **Первый** принимает авторизованные другими микросервисами запросы и печатает текст запроса заданное кол-во раз в
   консоль, после чего возвращает код 200 в ответном сообщении.
2. **Второй** возвращает то, что было напечатано в консоль в виде: пользователь + номер повторения + текст сообщения.

**Особенность:** сервис должен позволять делать паузы между повторами печати.

**Конфиг:** максимальное количество рабочих потоков должно задаваться через конфигурационный application-файл Spring.

### Модель, передаваемая в первый end-point:

* задержка печати на повторы в миллисекундах;
* имя пользователя в виде строки;
* кол-во повторов печати;
* текст, что надо напечатать.

### Требования к реализации:

Задачу решать в **Java 11**, **Spring Boot до 2.7**, используя **Thread** и **не синхронизированные коллекции**.
Подключить **Swagger** и описать по-русски представленный микросервисом API.

## Варианты решений

1. В ветке `master` лежит реализация, которая была показана на собеседовании. Решение неидеальное, по сути минимальный
   рабочий прототип, чтобы показать концепцию.
   > **Имеет серьезный недостаток:** если указать огромное число повторений, в памяти будет ОГРОМНАЯ очередь из
   объектов. 
2. В ветке `feature/another-impl` лежит версия, которая появилась в следствии моего желания сделать решение адекватнее.
