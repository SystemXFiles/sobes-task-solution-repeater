# sobes-task-solution-repeater

Решение задачи на собесе:
Жил был один маленький микросервис с двумя эндопинтами. 

1. Первый принимал авторизованные другими микросервисами запросы и печатал текст запроса заданное кол-во раз в консоль, после чего возвращал код 200 в ответном сообщении. 

2. Второй возвращал, то что было напечатано в консоль в виде: пользователь, который заказал печать, номер повторения печати указанного текста и текст который был напечатан. 

Особенность: умел этот микросервис производить временную задержку выполнения повтора печати для каждого пользователя.
Так же должна быть возможность ограничение потоков указывать через файл-конфигурационный в Spring.

Модель передаваемая в первый end-point:
  задержка печати на повторы в миллисекундах;
  имя пользователя в виде строки;
  кол-во повторов печати;
  текст, что надо напечатать.

Задачу решать в java 11 Spring Boot до 2.7 используя Thread и не синхронизированные коллекции. 
Подключить Swagger и описать по-русски представленный микросервисом API.
